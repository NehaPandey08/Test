Savings Calculator [React]
 INVEST NEW New React Web UI for the Savings Calculator on .co.uk.

  • Q5 product selection showing full catalogue (unauthenticated)

  • Tiered rate logic and reward product disclaimer handled client-side

  • Product Selector entry point pre-populates Q5 with recommended products
Covers S3 (authenticated search) and S4 (Product Selector).

xAPI Zone — Firewall
Savings Calculator xAPI
DAP API for Savings Calculator [Java]
 INVEST NEW New xAPI in the firewall zone. Single entry point for all three channels and all four scenarios.
  • Captures entry point context: search, Product Page, Product Selector, banner, deep link
  • Performs TIAA token exchange for authenticated journeys (S3, S4)
  • Orchestrates init calls: product catalogue + AEM content on all scenarios
  • S3 additionally calls Current and Savings Accounts Holdings Query API for held accounts
  • Returns single merged response to client. Zero API calls after init.
⚠ Dependency: Savings Products Catalogue Query API and new UKBA/AMS Product Enquiry sAPI (ISA data) must be available before end-to-end testing.
TIAA
Trusted Identity and Attribute Authority
 INVEST RE-USE Re-used for token exchange on S3 and S4 authenticated journeys. No changes. Not called for S1 and S2.
Journey Registry xAPI
Digital Journey Registry Management [Java]
 INVEST RE-USE Re-used for session registration on all scenarios. Entry point context captured and registered on session initialisation. No changes required.
Campus Zone — Product Data
AEM API Gateway
AEM Gateway Webweaver [GraphQL]
 INVEST RE-USE Re-used to serve calculator content on initialisation: question labels, tooltips, disclaimers, validation messages. Called on every scenario. No changes to the gateway.
AEM
Adobe Experience Manager [Adobe CMS]
 INVEST AMEND New content components to be authored for the Savings Calculator:
  • Five question labels and tooltip text
  • Variable rate change disclaimer and reward product withdrawal disclaimer
  • ISA disclaimer
  • Validation messages and information icon content
No changes to the AEM platform.
Savings Products Catalogue Query API
Domain API — product data [Java] — New
 INVEST NEW New Domain API in the campus zone. Normalises variable rate and fixed rate product data into a common product schema. Two endpoints:
  • Savings Product List GET — full catalogue with AER per product type. Used by S1 and S3.
  • Savings Product Detail GET — accepts one or more product codes. Returns product details and AER. Used by S2 (single code) and S4 (multiple codes from Product Selector).
Uses Ehcache internally to reduce load on upstream sAPIs. Reusable beyond the calculator. Per ADR-CALC-001.
⚠ Dependency on new UKBA/AMS Product Enquiry sAPI for ISA product data.
Campus Zone — Account Data (S3 Only)
Current and Savings Accounts Holdings Query API
[Java, Spring Boot]
 INVEST RE-USE Re-used from Bond Maturity (live). Retrieves all held savings accounts for authenticated customers including variable rate, bonds, and ISAs with current balances. Fixed rate product names retrieved from PSOS via profileMarker 127.
  • Called by the Savings Calculator xAPI as the third init call for S3 only
  • Parameters: closedAccount=N (open accounts only)
  • Internally calls ODS Retail Product Holdings & Account Data sAPI, ODS Retail Account Enquiry sAPI, PSOS
ODS / sAPI Zone
Static Data Hub sAPI
Product reference data [Java]
 INVEST RE-USE Re-used to retrieve variable rate product data from SDH. Called by the Savings Products Catalogue Query API for all scenarios.
  • Variable rate products only: AER, gross rate, tier config, reward flag
  • Bonds and ISAs are not sourced from SDH — covered by new UKBA/AMS Product Enquiry sAPI
Per ADR-CALC-001. No changes required.
New UKBA/AMS Product Enquiry sAPI
Fixed rate product data by product code — New
 INVEST NEW New sAPI required to retrieve fixed rate product-level data from UKBA/AMS by product code. Covers both bonds and ISAs. No existing sAPI returns product-level data including AER by product code without a customer account ID.
  • Must return: AER, gross rate, tier configuration, interest payment type, product attributes
  • Covers bonds (AMS-UKBA) and ISAs (UKBA)
  • No customer context required — product-level query only
⚠ Hard blocker for all fixed rate product display. Must be built by UKBA/AMS team. On critical path for all four scenarios.
ODS Retail Product Holdings & Account Data sAPI
[Java]
 INVEST RE-USE Re-used internally by Current and Savings Accounts Holdings Query API for S3. Retrieves customer product holdings from Customer ODS. No direct call from the calculator. No changes required.
ODS Retail Account Enquiry sAPI
[Java]
 INVEST RE-USE Re-used internally by Current and Savings Accounts Holdings Query API for S3. Called in batches of 15 account IDs to retrieve account balance. No changes required.
ODS Retail Customer Enquiry sAPI
[Java]
 INVEST RE-USE Re-used internally by Current and Savings Accounts Holdings Query API for S3. Retrieves customer markers and address data. No changes required.
