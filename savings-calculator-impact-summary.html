xAPI ZONE — FIREWALL
10
Savings Calculator xAPI
DAP API for Savings Calculator
[Container: Java] — New
 NEW INVEST A new xAPI is required sitting in the firewall zone as the single entry point for all three channels. Responsibilities include capturing entry point context (search, Product Page, Product Selector, banner, deep link), performing TIAA token exchange for authenticated journeys (S3, S4), orchestrating init calls to the Savings Products Catalogue Query API and AEM API Gateway, and for S3 also calling the Current and Savings Accounts Holdings Query API. Returns a single merged response to the client on initialisation. No further API calls are made after init. All four scenarios are served by this single xAPI.
Dependency: Savings Products Catalogue Query API (new domain API) and new UKBA/AMS Product Enquiry sAPI must be available before end-to-end testing.

CAMPUS ZONE — PRODUCT DATA
13
Savings Products Catalogue Query API
Domain API — product data
[Container: Java] — New
 NEW INVEST A new Domain API is required in the campus zone. It is the single source of normalised product data for the calculator, aggregating variable rate data from SDH and fixed rate data from the new UKBA/AMS Product Enquiry sAPI into a common product schema. Exposes two endpoints: Savings Product List GET (full catalogue, S1 and S3) and Savings Product Detail GET (one or more product codes, S2 and S4). Uses Ehcache internally to reduce load on upstream sAPIs. Reusable beyond the calculator. Per ADR-CALC-001.
Dependency: New UKBA/AMS Product Enquiry sAPI required for fixed rate product data. This is on the critical path.
 Calculator Team

19
New UKBA/AMS Product Enquiry sAPI
Fixed rate product data by product code
[New — to be built by UKBA/AMS team]
 NEW INVEST A new sAPI is required to retrieve fixed rate product data from UKBA/AMS by product code. No existing sAPI provides product-level data (AER, gross rate, tier configuration, interest payment type) for fixed rate products without a customer account ID. The existing ZOS UKBA ISA Enquiry sAPI and ZOS UKBA Bonds Enquiry sAPI are account-level APIs and cannot be used for this purpose. The new sAPI must return all product-level attributes needed by the Savings Products Catalogue Query API to build the common product schema.
⚠ Hard blocker. This sAPI does not exist. Must be built by the UKBA/AMS team. Without it, fixed rate bonds and ISAs cannot be displayed in the calculator. This is on the critical path for all four scenarios and must be raised with the UKBA/AMS team as a priority dependency.

Critical Dependencies and Blockers
New UKBA/AMS Product Enquiry sAPI Does not exist. Must be built by the UKBA/AMS team. Hard blocker for fixed rate product display across all scenarios. Must be raised immediately.
Product Hub (future) When Product Hub goes live, only the Savings Products Catalogue Query API changes its data source. No impact on xAPI or channel clients. A separate ADR will be raised at that point. ADR-CALC-001.
Q1 pre-population Whether selecting a held account at Q5 auto-populates Q1 with the account balance is pending BA confirmation. This will impact calculation engine behaviour and client-side state management.
S2 existing API check Check whether an existing API can serve the product detail GET endpoint for S2 before building a new one.
